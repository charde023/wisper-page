# TechBridge-KR 학습노트 자동화 파이프라인 기획서

> 목표: **유튜브 채널 감시 → 전사 → LLM 교정 → 옵시디언 학습노트 → GitHub Pages 발행**을 맥에서 **무인 자동화**.
> 전제: 전사 파이프라인(mlx-whisper)은 [2026-07-02_맥-로컬-전사-파이프라인 구현]로 이미 완성.

---

## 0. 결론 (확정 사항 반영)

| 축 | 결정 |
|---|---|
| **전사** | mlx-whisper `large-v3-turbo` (구축 완료, Metal GPU, 26~45배속) |
| **교정(transcript_clean)** | **코덱스 프록시 `gpt-5.4-mini`** (localhost:18080, 종량0) — 분리형 스텝 신설 |
| **노트화** | **코덱스 프록시 `gpt-5.5`** (한국어 품질) — 소넷 수동작성 대체 |
| **발행** | GitHub Pages `charde023/page` 의 **`study-notes/` 그룹**(기존 랜딩과 격리) |
| **큐레이션** | **느슨 필터** — 명백 저가치(점수 바닥권)만 컷, 웬만하면 노트화 |
| **스케줄** | 맥 **launchd** 매일 밤 23시 |
| **금지** | ❌ headless `claude -p`(유료 종량) — 절대 안 씀 |

핵심 = **LLM이 필요한 두 지점(교정·노트화)을 전부 이미 가동 중인 코덱스 프록시(종량0)로 배선**하고, 전사·발행·감시를 launchd 오케스트레이터로 엮어 손 안 대고 돌아가게 한다.

---

## 1. 배경 — 지금 무엇이 수동인가

방금 완성한 파이프라인은 6단계 중 **④ 노트화가 수동**(소넷 에이전트 수동 디스패치)이고, **교정이 노트화에 묻혀 있고**, **발행(HTML)·감시(스케줄) 단계가 없다**. 이 4개를 채우면 완전 무인이 된다.

| 단계 | 현재 | 자동화 후 |
|---|---|---|
| ① 감시 | 수동 `rss_watch.py` | launchd 야간 자동 |
| ② 큐레이션 | 수동 판단 | `curate.py` 느슨 필터 |
| ③ 전사 | ✅ `run_youtube.py` | 그대로 |
| ④ 교정 | 노트화에 묻힘 | **분리 — `transcript_clean.py`(gpt-5.4-mini)** |
| ⑤ 노트화 | 수동 소넷 | **`author_note.py`(gpt-5.5)** |
| ⑥ 인덱스·커밋 | 수동 | 오케스트레이터 자동 |
| ⑦ 발행 | 없음 | **`publish_study_notes.py` → page/study-notes/** |
| ⑧ 알림 | 없음 | 데스크톱/슬랙 요약 |

## 2. 목표 아키텍처

```
launchd (매일 밤 23시)
   └─ nightly.py (오케스트레이터)
        ① rss_watch.py --json         신규 영상 감지
        ② curate 느슨 필터            저가치 컷 (전사만/스킵)
        ③ run_youtube.py              페치 + mlx-whisper 전사
        ④ transcript_clean.py         [코덱스 gpt-5.4-mini] ASR 오인식·고유명사 교정
        ⑤ author_note.py              [코덱스 gpt-5.5] 한국어 학습노트 MD 생성 → 볼트
        ⑥ rebuild_index.py + git      목차 갱신 + 볼트 커밋·푸쉬
        ⑦ publish_study_notes.py      MD→HTML → page/study-notes/ 발행·푸쉬
        ⑧ notify                      완료 요약(성공/실패 건수·링크)
```

**엔진 배선 = 코덱스 프록시 (이미 launchd로 상시 가동)**
- 엔드포인트: `http://127.0.0.1:18080/v1` (OpenAI 호환)
- 모델: `gpt-5.4-mini`(교정) · `gpt-5.5`(노트화) — ChatGPT 구독 경유 **종량0**
- 호출 = OpenAI SDK에서 `base_url`만 이걸로. api_key는 더미(프록시가 `~/.codex` auth 사용)
- 프록시 죽으면 launchd가 재기동. 오케스트레이터는 **시작 전 `/v1/models` 헬스체크**.

## 3. 파트 A — GitHub Pages 학습노트 발행 (기존과 격리)

**현황**: `charde023/page` = 루트 `index.html`(지피터스 랜딩) + 그룹 폴더(`gpters/`·`reports/`·`apom/`·`회의록/`)가 각자 `index.html`. 개별 페이지 = `slug/index.html`.

**설계**: 같은 패턴으로 **`study-notes/` 그룹 신설**.

| 경로 | 역할 | URL |
|---|---|---|
| `page/study-notes/index.html` | 학습노트 전용 랜딩(카드·최신순·채널/요약/태그) | `…/page/study-notes/` |
| `page/study-notes/<slug>/index.html` | 노트 1개 = HTML 1장(자체완결) | `…/study-notes/<slug>/` |
| 루트 `index.html` | **안 건드림**(지피터스 유지) → 완전 격리 | — |

**옵시디언 → HTML 변환 규칙** (우리 노트 특유 문법)
| 옵시디언 | HTML |
|---|---|
| `> [!summary]` / `> [!info]` | 색상 콜아웃 박스 |
| `> [!note]- 전체 전사 펼치기` | `<details><summary>` 접기 |
| `<span style="color:#ef6c00">` | 그대로(주황) |
| `[[위키링크]]` | study-notes 내부 존재 시 링크, 아니면 텍스트 |
| frontmatter | 상단 메타 카드(원작자·소속·업로드·길이·원본URL) |

**스타일**: 기존 page 디자인 토큰(Pretendard·820px·`word-break:keep-all`) 재사용 → 사이트 통일.
**초기 발행**: 이미 만든 6개 → study-notes/ 시딩.

## 4. 파트 B — 감시→끝까지 자동화 (상세)

### 4.1 큐레이션 (느슨 필터)
`curate.py` 점수 `2.0·토픽 + 1.6·개념 + 4.0·화자 + 0.8·조회백분위`.
- **느슨 규칙**: `score ≥ 바닥임계(예 1.0)` 이면 노트화, 그 미만만 컷.
- 컷된 것도 **전사·발행은 진행하되 노트는 생략**(원하면 "전사만" 모드) — 방침은 "명백 저가치만 컷"이라 대부분 통과.
- 임계값은 파일럿에서 6개 실제 점수 보고 확정.

### 4.2 교정 (transcript_clean, gpt-5.4-mini)
raw 전사의 ASR 오류를 문맥 교정. **`youtube.json`의 description/제목/링크를 고유명사 힌트로 프롬프트에 주입**(예: `Honk`·`Niklas Gustavsson`·`Cloud→Claude`). 산출 = `transcript_clean.txt`(정제 영어). → 노트화·HTML 접기 전사가 이 clean본 사용.

### 4.3 노트화 (author_note, gpt-5.5)
`transcript_clean.txt` + `youtube.json` + `note_template.md` → 한국어 학습노트 MD. **볼트 규칙 프롬프트에 고정**(이모지·`==하이라이트==` 금지, 강조 2종, frontmatter, 접기식 전사). → `학습노트/TechBridge-KR/<제목>.md`.

### 4.4 스케줄·멱등·복구
- **launchd** `kr.techbridge.nightly` 매일 23시(앱 불필요 CLI). 지터 허용.
- **멱등**: 각 단계 산출물 존재 시 스킵(전사·교정·노트·발행 각각). seen 마킹은 **발행 성공 후에만** → 실패 시 다음 밤 재개.
- **복구**: 페치 일시실패 자동 재시도, 프록시 다운 시 헬스체크 후 대기/알림.
- **알림**: 완료 요약(신규 n·노트 n·발행 링크·실패 사유).
